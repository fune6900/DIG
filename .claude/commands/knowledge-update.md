蒐集のメイド（sub-agent-knowledge）を起動し、ヴィンテージ古着のナレッジ情報をMarkdown形式でローカルに保存し、NotionのナレッジベースページにもMCPで同期する。

## 引数

引数なしで起動した場合はマスターに以下を対話形式で確認する:

- ブランド名（例: Levi's, Champion, Carhartt）
- 対象アイテム（例: 501, リバースウィーブ, デトロイトジャケット）
- 年代（例: 1950s〜1960s, 1970年代後半）
- ディテール情報（タグ/ジッパー/ボタン/ステッチの特徴）
- 相場情報（任意）
- 参考URL（任意）

## 手順

### Phase 1: 情報収集

引数またはマスターへの確認で以下の情報を収集する:

```
brand: <ブランド名>
item: <アイテム名>
era: <年代>
details:
  - type: タグ | ジッパー | ボタン | ステッチ | その他
    partName: <パーツ名（例: TALON, ドーナツボタン）>
    description: <特徴の説明>
price_range: <相場（任意）>
notes: <補足情報（任意）>
```

### Phase 2: Markdownファイルへの保存

1. ブランド名を kebab-case に変換（例: "Levi's" → "levis"）
2. `.claude/knowledge-base/brands/<ブランド名>.md` を確認する
   - ファイルが存在しない場合: 新規作成
   - 存在する場合: 末尾に追記
3. 以下のフォーマットで保存する:

```markdown
## <アイテム名> — <年代>

**ブランド**: <ブランド名>
**年代**: <年代>
**更新日**: YYYY-MM-DD

### 識別ポイント

| 種別     | パーツ名   | 特徴   |
| -------- | ---------- | ------ |
| タグ     | -          | <説明> |
| ジッパー | <パーツ名> | <説明> |

### 相場

<相場情報（任意）>

### 補足

<補足情報（任意）>

---
```

### Phase 3: Notion へのページ作成

Notion MCP を使って「古着のナレッジ倉庫」データベースに **1ナレッジ1ページ** で追加する。

**同期先データソースID**: 環境変数 `NOTION_KNOWLEDGE_DATA_SOURCE_ID`
**データベースURL**: 環境変数 `NOTION_KNOWLEDGE_DATABASE_URL`
**格納ページURL**: 環境変数 `NOTION_KNOWLEDGE_PARENT_PAGE_URL`

1. `mcp__notion__notion-create-pages` で新規ページを作成する
2. `parent` には `data_source_id` として環境変数 `NOTION_KNOWLEDGE_DATA_SOURCE_ID` の値を指定する
3. `properties` に `名前` を以下の形式で設定する:
   ```
   <ブランド名> — <アイテム名>（<年代>）
   ```
4. `content` にナレッジの詳細を以下の形式で記述する:

```markdown
**ブランド**: <ブランド名>
**年代**: <年代>
**更新日**: YYYY-MM-DD

## 識別ポイント

| 種別   | パーツ名   | 特徴   |
| ------ | ---------- | ------ |
| <種別> | <パーツ名> | <説明> |

## 相場

<相場情報（省略可）>

## 補足

<補足情報（省略可）>
```

### Phase 4: 完了報告

以下を報告する:

- 保存したローカルファイルパス
- 作成した Notion ページの URL
- ページタイトルと識別ポイントの要約

## 注意

- 情報の信頼性をマスターに確認してから保存する
- 同名エントリが既に存在する場合は新規作成せずマスターに確認する
- Notion への作成先は常に環境変数 `NOTION_KNOWLEDGE_DATA_SOURCE_ID` の値を使用する
- 環境変数が未設定の場合はマスターに設定を依頼してから実行する
